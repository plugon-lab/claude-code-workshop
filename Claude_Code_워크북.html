📋 사전 준비 (강의 전에 완료)

설치는 강의 중 함께 진행합니다. 아래 두 가지만 미리 준비해 오세요.

1
1-1. VS Code 설치
  1. code.visualstudio.com 접속
  2. Download for Windows 버튼 클릭 → .exe 파일 다운로드
  3. 다운로드된 파일 실행 → 설치 옵션은 기본값 유지
  4. 설치 중 "PATH에 추가" 옵션이 있으면 반드시 체크
  5. 설치 완료 후 VS Code 실행 확인
  1. code.visualstudio.com 접속
  2. Download for Mac 버튼 클릭 → .zip 파일 다운로드
  3. 다운로드된 .zip 더블클릭 → Visual Studio Code.app 압축 해제
  4. Visual Studio Code.app을 응용 프로그램(Applications) 폴더로 드래그
  5. VS Code 실행 후 상단 메뉴 → Shell Command: Install 'code' command in PATH 실행
마지막 PATH 등록이 중요합니다. 나중에 터미널에서 code . 명령으로 폴더를 VS Code로 바로 열 수 있습니다.

  4. 설치 옵션은 모두 기본값 → Next 계속 클릭
  5. 설치 완료 후 PowerShell 또는 명령 프롬프트 새로 열기
  6. 아래 명령으로 설치 확인:
PowerShell

→ 만들어진 파일을 VS Code 탐색기(왼쪽 파일 목록)에서 확인해보세요.

count.py에서 짝수만 출력하도록 수정해줘

"이거 수정해줘", "이거 추가해줘"처럼 대화하듯 이어가는 게 Claude Code의 핵심 사용법입니다.

PART 3. Playwright MCP 실습
60분 · MCP 개념 이해 + 브라우저 자동화 실습
1
3-1. MCP란? (이론 10분)

MCP는 Claude Code에게 외부 도구를 연결하는 플러그인 시스템입니다.

MCP 구조
Claude Code
  ├── 기본: 파일 읽기/쓰기, 터미널 실행
  └── MCP 연결 시:
        ├── Playwright  → 브라우저 자동화
        ├── Notion      → 노션 문서 읽기/쓰기
        ├── Google Drive → 구글 드라이브 연동
        └── 기타 수백 가지 도구...

Playwright MCP를 연결하면 Claude Code가 직접 브라우저를 열고, 클릭하고, 스크린샷을 찍을 수 있습니다.

2
3-2. Playwright MCP 설치 및 설정

① 패키지 설치 (공통)

bash 
npm install -g @playwright/mcp
npx playwright install chromium

② settings.json 설정

Ctrl+Shift+P → Open User Settings (JSON)

또는 직접 경로: C:\Users\사용자명\.claude\settings.json

Cmd+Shift+P → Open User Settings (JSON)

또는 직접 경로: ~/.claude/settings.json

settings.json 
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
파일에 다른 내용이 이미 있다면 mcpServers 부분만 추가하세요.

③ Claude Code 재시작

bash 
claude
3
3-3. Playwright MCP 실습
실습 1 — 브라우저 열고 스크린샷
playwright를 사용해서 구글 홈페이지를 열고 스크린샷을 찍어줘. 스크린샷 파일은 screenshot_google.png로 저장해줘.
실습 2 — 검색 자동화
playwright로 네이버(naver.com)에 접속해서 검색창에 "Claude Code 강의"를 입력하고 검색한 뒤 결과 페이지 스크린샷을 screenshot_search.png로 저장해줘.
실습 3 — 링크트리 자동 검수 (Windows)
playwright를 사용해서 내 링크트리 페이지를 열고 검수해줘. 파일 경로는 C:\Users\사용자명\Documents\my-linktree\index.html 이야. 확인해야 할 것들: 1. 페이지가 정상적으로 열리는지 2. 프로필 영역이 보이는지 3. 링크 버튼 5개가 모두 보이는지 4. 모바일 크기(390px)로 변경했을 때도 잘 보이는지 각 단계별로 스크린샷을 찍고, 발견된 문제가 있으면 알려줘.
실습 3 — 링크트리 자동 검수 (Mac)
playwright를 사용해서 내 링크트리 페이지를 열고 검수해줘. 파일 경로는 /Users/사용자명/Documents/my-linktree/index.html 이야. 확인해야 할 것들: 1. 페이지가 정상적으로 열리는지 2. 프로필 영역이 보이는지 3. 링크 버튼 5개가 모두 보이는지 4. 모바일 크기(390px)로 변경했을 때도 잘 보이는지 각 단계별로 스크린샷을 찍고, 발견된 문제가 있으면 알려줘.
내 정확한 경로 확인: 터미널에서 my-linktree 폴더 안에서 pwd 실행
실습 4 — 문제 자동 수정 (도전 과제)
방금 검수에서 발견된 문제들을 수정해줘. 수정 후 다시 playwright로 확인해줘.
MCP를 통해 Claude Code가 "눈"을 갖게 됩니다. 코드 작성 → 브라우저 확인 → 문제 발견 → 수정 사이클을 자동으로 돌릴 수 있습니다.

🔧 문제 해결 (트러블슈팅)

Claude Code 명령을 못 찾는 경우

bash 
npm list -g @anthropic-ai/claude-code
npm uninstall -g @anthropic-ai/claude-code
npm install -g @anthropic-ai/claude-code

PowerShell을 관리자 권한으로 다시 실행한 뒤 재설치하세요.

zsh PATH 추가 
echo 'export PATH="$PATH:$(npm prefix -g)/bin"' >> ~/.zshrc
source ~/.zshrc

로그인 오류가 나는 경우

bash 
claude logout
claude
Free 플랜이면 로그인 후 오류가 발생합니다. claude.ai에서 Pro/Max 플랜 여부를 확인하세요.

Playwright MCP가 연결 안 되는 경우

  • settings.json 경로 확인 — Windows: C:\Users\사용자명\.claude\settings.json / Mac: ~/.claude/settings.json
  • Claude Code 완전 종료 후 재시작
  • npx playwright install chromium 재실행
  • Mac 보안 경고 시: 시스템 설정 → 개인 정보 보호 및 보안 → Chromium 실행 허용

GitHub push 오류가 나는 경우

bash 
git remote -v
git remote set-url origin https://github.com/내아이디/my-linktree.git

강의 관련 문의는 강사에게 직접 연락하세요.
